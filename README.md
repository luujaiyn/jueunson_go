> 디렉토리 구성 생각 설명
- cmd/server/main.go
실행 진입점을 따로 두게 서버 시작, 라우터 생성, 핸들러 연결만 담당하도록 분리

- internal/handler/
인증, 사용자, 금융, 게시판처럼 HTTP 요청/응답 처리 로직을 기능별로 분리

- internal/model/
요청/응답 구조체와 공통 타입을 따로 분리

- internal/session/
로그인 세션과 토큰 관리하는 부분 깔끔하게 따로 분리

- internal/store/
사용자, 게시글, 금융 관련 DB 접근 로직을 모아 분리

- internal/web/
쿠키 처리, 토큰 추출, 사용자 응답 가공, 정적 파일 등록 같은 웹 보조 기능을 따로 분리

- static/
HTML, CSS, JS 등 프론트 자원

- schema.sql, seed.sql, app.db
DB 구조 정의, 초기 데이터, 실제 DB 파일을 코드와 분리

-> main.go 설명 (보안적으로 고려를 한 부분)

- 인증 없는 요청이 API에 들어오는 걸 막기 위해 토큰 추출 후 세션 검증을 적용해 방어했다. 로그인하지 않은 사용자가 /me, 게시판, 금융 기능에 접근하는 걸 막기 위해 토큰 확인 + 세션 조회를 공통적으로 적용했음

- 클라이언트가 사용자 정보를 조작해서 다른 사람 계정으로 입출금이나 게시글 작성하는 걸 막기 위해 세션 사용자 기준으로 처리하도록 구성

- 세션 토큰이 클라이언트 스크립트에서 직접 노출되는 걸 막기 위해 HttpOnly 쿠키와 SameSite 설정을 적용해 방어

- 로그인된 상태만으로 민감한 회원탈퇴가 실행되는 걸 막기 위해 탈퇴 시 비밀번호를 다시 확인하도록 구성

- 송금/출금 중간에 잔액 불일치나 부분 반영이 생기는 걸 막기 위해 트랜잭션 기반으로 처리하도록 구성

- 잘못된 입력값이 그대로 처리되는 것을 막기 위해 모든 요청에 대해 JSON 바인딩 검증과 필수값 검사, 공백 제거를 적용했음

- 없는 사용자나 게시글에 대한 요청이 정상 처리되는 것을 막기 위해, DB 조회 시 대상 존재 여부를 확인하고 결과에 따라 분기 처리하도록 구성

- 금액이 0보다 적어지는 것 같은 마이너스나 비정상 금액이 처리되는 것을 막기 위해 입금, 출금, 송금 시 금액이 양수인지 검증하도록 구성했음.
